Exercise 1 - Lasso Performance with Different Sample Sizes

We define True Positives (TP) as non-zero coefficients that have correctly been selected and False Positive (FP) as zero coefficients that have been selected. The “Precision” parameter is a ratio of TP and the total selected parameters, so it gives a measure of how good the model is at selecting the right coefficients. The “Recall” parameter is the ratio of TP and the number of non-zero coefficients (TP+False Negatives). We expect a lower precision and recall for smaller sample sizes, as the training data is smaller. As sample size increases there should be a convergence towards 1.00. The noise was set to 10.0 to add a more realistic feature and the regularization strength was set to 1.00 to increase the penalization.

# Import libraries
import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import Lasso
from sklearn.metrics import precision_score, recall_score

# Set random seed for reproducibility
np.random.seed(42)

# Set parameters
n_features = 100
n_informative = 10
alph = 1.0  # Regularization strength
n_samples_list = [100, 200, 1000]  # Sample sizes

precisions = []
recalls = []

# Loop over different sample sizes
for n_samples in n_samples_list:
    # Generate feature matrix X
    X = np.random.randn(n_samples, n_features)

    # Create sparse true coefficients
    coef_true = np.zeros(n_features)
    nonzero_idx = np.random.choice(n_features, n_informative, replace=False)
    coef_true[nonzero_idx] = np.random.uniform(1, 5, size=n_informative)

    # Generate target variable y with noise
    y = X @ coef_true + np.random.normal(0, 10, size=n_samples)

    # Fit Lasso model
    lasso = Lasso(alpha=alph)
    lasso.fit(X, y)

    # Get selected features (non-zero coefficients)
    selected = lasso.coef_ != 0
    true_nonzero = coef_true != 0

    # Compute precision and recall
    precision = precision_score(true_nonzero, selected)
    recall = recall_score(true_nonzero, selected)

    precisions.append(precision)
    recalls.append(recall)

# Plot precision and recall
plt.figure(figsize=(8,5))
plt.plot(n_samples_list, precisions, label="Precision", marker='o')
plt.plot(n_samples_list, recalls, label="Recall", marker='o')
plt.xlabel("Sample Size")
plt.ylabel("Score")
plt.title("Lasso Feature Selection Performance vs Sample Size")
plt.ylim(0, 1.05)
plt.legend()
plt.grid(True)
plt.show()

The plot is coherent with what we expected, showing a clear upward trend in Precision and Recall parameters. It is worth noting that, since Precision is lower than 1, there are still some False Positives. Recall is, instead, precisely 1.00, meaning that there are no False Negatives.

Exercise 2 - Lasso performance with different sparsity levels

# Set parameters
np.random.seed(42)
n_features = 100
alph = 1.0
sample_size = 100  # Fixed sample size
n_informative_list = [5, 20, 50, 100]  # Varying sparsity levels

precisions = []
recalls = []

# Loop over different numbers of informative variables
for n_informative in n_informative_list:
    # Generate feature matrix X
    X = np.random.randn(sample_size, n_features)

    # Create sparse true coefficients
    coef_true = np.zeros(n_features)
    nonzero_idx = np.random.choice(n_features, n_informative, replace=False)
    coef_true[nonzero_idx] = np.random.uniform(1, 5, size=n_informative)

    # Generate target variable y with noise
    y = X @ coef_true + np.random.normal(0, 10, size=sample_size)

    # Fit Lasso model
    lasso = Lasso(alpha=alph, max_iter=10000)
    lasso.fit(X, y)

    # Evaluate model
    selected = lasso.coef_ != 0
    true_nonzero = coef_true != 0
    precision = precision_score(true_nonzero, selected)
    recall = recall_score(true_nonzero, selected)

    precisions.append(precision)
    recalls.append(recall)

# Plotting
plt.figure(figsize=(8,5))
plt.plot(n_informative_list, precisions, label="Precision", marker='o')
plt.plot(n_informative_list, recalls, label="Recall", marker='o')
plt.xlabel("Number of Informative Features (Sparsity)")
plt.ylabel("Score")
plt.title("Lasso Feature Selection Performance vs Sparsity")
plt.ylim(0, 1.05)
plt.legend()
plt.grid(True)
plt.show()

As expected, Lasso chooses the right parameter more easily by reducing sparsity (increasing the number of relevant parameters). On the other hand, Recall decreases, meaning that it becomes harder for the model to choose all of the relevant parameters. This means that there are two scenarios: 1. In sparse frameworks (few relevant coefficients) precision is low but recall is high. 2. In less sparse frameworks (more relevant coefficients) precision is high but recall is low.

Exercise 3 - Lasso Performance with Multicollinearity

# Import again in case this block is run independently
import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import Lasso
from sklearn.metrics import precision_score, recall_score

np.random.seed(42)

# Set constants
n_features = 100
n_informative = 10
alph = 1.0
n_samples = 200

# Test different levels of correlation between features
corr_strengths = np.linspace(0.0, 0.95, 15)

precisions = []
recalls = []

for corr_strength in corr_strengths:
    # Create correlation matrix
    correlation_matrix = np.eye(n_features) + corr_strength * (np.ones((n_features, n_features)) - np.eye(n_features))

    # Ensure it's positive semi-definite
    eigvals = np.linalg.eigvalsh(correlation_matrix)
    if np.any(eigvals <= 0):
        raise ValueError("Correlation matrix is not positive semi-definite")

    # Generate multicollinear features
    mean = np.zeros(n_features)
    X = np.random.multivariate_normal(mean, correlation_matrix, size=n_samples)

    # Sparse true coefficients
    coef_true = np.zeros(n_features)
    nonzero_idx = np.random.choice(n_features, n_informative, replace=False)
    coef_true[nonzero_idx] = np.random.uniform(1, 5, size=n_informative)

    # Generate y
    y = X @ coef_true + np.random.normal(0, 10, size=n_samples)

    # Fit model
    lasso = Lasso(alpha=alph)
    lasso.fit(X, y)

    # Compute scores
    selected = lasso.coef_ != 0
    true_nonzero = coef_true != 0
    precision = precision_score(true_nonzero, selected)
    recall = recall_score(true_nonzero, selected)

    precisions.append(precision)
    recalls.append(recall)

# Plotting
plt.figure(figsize=(8, 5))
plt.plot(corr_strengths, precisions, label="Precision", marker='o')
plt.plot(corr_strengths, recalls, label="Recall", marker='o')
plt.xlabel("Feature Correlation Strength")
plt.ylabel("Score")
plt.title("Lasso Feature Selection vs Feature Correlation")
plt.ylim(0, 1.05)
plt.xticks(corr_strengths, [f"{x:.2f}" for x in corr_strengths])
plt.legend()
plt.grid(True)
plt.show()

Both Recall and Precision appear to be pretty stationary up to 0.6 of correlation strength. After that point they start to decrease but both have a spike at 0.8. Lasso is good at low levels of correlation, being able to find almost all relevant coefficients (Recall is almost 1). It still selects a bunch of non-relevant coefficients, since Precision starts at a low value of 0.4. As correlation increases both measures show a decline (very evident in the ability to recall the right coefficients).

Exercise 4 - Interpretation

Methodological differences

The difference between Lasso (Least Absolute Shrinkage and Selection Operator) and Ridge feature selection models is based on the different penalization terms used. Lasso adds an L1 penalty to the loss function that uses the sum of the absolute values of coefficients. Ridge adds an L2 penalty that is based on the sum of squared coefficients.

Impact on the coefficients and benefits of each method

Lasso forces some coefficients to be 0, thus making a selection, improving the interpretability and reducing overfitting (particularly in models with many dimensions). Ridge does not force coefficients to 0 but it shrinks them down towards it, making this method more suited when it is crucial to preserve the contribution of each coefficient (even if small).

Limitations of Lasso

In high correlation frameworks Lasso tends to lose the ability to recall the right coefficients. As the previous plot showed increasing the strength of correlation leads to a decrease in the ability to recall the right coefficients. This is why another approach might be more suited, namely Ridge or Elastic Net. To address the correlation issue, in particularly complex situations, a combination of L1 and L2 penalties (Elastic Net) might be adopted to have a result that is both precise and easy to interpret.
